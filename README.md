## Кратко про новый текстовый движок (чуть ниже список функций)

Новый текстовый движок использует синтаксис через `\`, а не через классические `#function#`.

Функции с аргументами пишутся через скобки:

```text
\function(...)
```

Функции без аргументов могут завершаться символом `^`:

```text
\tick^
```

В этом случае `^` — это не степень, а завершение функции без параметров.

Математические операции в новом движке тоже выглядят как функции:

| Функция | Описание |
|---|---|
| `\+(a b)` | Сложение. |
| `\-(a b)` | Вычитание. |
| `\*(a b)` | Умножение. |
| `\/(a b)` | Деление. |
| `\mod(a b)` | Остаток от деления. |

Аргументы разделяются пробелом:

```text
\/(\tick^ 1000)
```

Здесь `\tick^` делится на `1000`.

Функции можно вкладывать друг в друга через скобки:

```text
\timef(\/(\tick^ 1000))
```

Выполнение идёт изнутри наружу:

1. `\tick^` — получить текущий tick.
2. `\/(\tick^ 1000)` — разделить tick на `1000`.
3. `\timef(...)` — отформатировать результат.

В отличие от классического движка, здесь не нужны обратные кавычки `` ` ``, которые часто используются в классическом движке для вложенных функций.

Обычный текст можно писать рядом с функциями:

```text
Время: \timef(\/(\tick^ 1000))
```

Пример преобразования значения в формат с миллисекундами:

```text
\timef(\/(\tick^ 1000)):\mod(\tick^ 1000)
```

Разбор:

1. `\tick^` — возвращает текущее значение tick в миллисекундах.
2. `\/(\tick^ 1000)` — делит миллисекунды на `1000`, получая секунды.
3. `\timef(\/(\tick^ 1000))` — форматирует секунды в читаемый вид.
4. `\mod(\tick^ 1000)` — возвращает остаток миллисекунд в текущей секунде.

Итоговый формат:

```text
минуты:секунды:миллисекунды
```

# Классический текстовый движок

Дальше идут функции и обозначения для классического текстового движка, где используется синтаксис `#function#`.

## Условные обозначения

- `*playerid` — необязательный параметр игрока.
- Если необязательный параметр указывается после других параметров, важно соблюдать запятые/разделители как в синтаксисе функции.
- `#retval(id)#` — числовое значение из коллбэка или события.
- `#retstr(id)#` — строковое значение из коллбэка или события.
- `N/A` — значение не используется или не определено.
- `0/1` — обычно булевый статус: `0` = нет/выключено/закрыто, `1` = да/включено/открыто.

# Текстовые функции

## Игрок, мир, переменные и базовые значения

| Функция | Описание |
|---|---|
| `#playerid#` | Возвращает ID текущего игрока. Может принимать глобальный ID: например, `#playerid(2)#` вернёт ID игрока с глобальным ID 2, если он онлайн. |

| `#array(slot, *playerid)#` | Возвращает массив игрока. Слот: `0–26`. `*playerid` необязателен. |
| `#server(slot)#` | Возвращает массив сервера. Слот: `0–49`. |
| `#var(name)#` | Возвращает переменную мира из `/varlist`. |
| `#pvar(name, *playerid)#` | Возвращает переменную игрока из `/pvarlist`. |
| `#teamOnline(teamid)#` | Возвращает онлайн конкретной команды по `teamid`. |
| `#online#` | Возвращает онлайн мира. |
| `#timestamp#` | Возвращает время в секундах. |
| `#time#` | Возвращает время мира. |
| `#weather#` | Возвращает погоду мира. |
| `#admin#` | Возвращает серверный статус администратора. |
| `#tick#` | Возвращает количество тиков с момента запуска сервера. В теме указано: 1 тик = 1 мс. |
| `#worldName#` | Возвращает название мира. |
| `#worldDesc#` | Возвращает описание мира. |

- `*` — необязательное значение.
- `**` — значение необязательно, если не используется `cond`.

Категории:

